# 請求的使用

## 基本用法

```python
yield scrapy.Request(
    url="https://example.com",
    method="get", # 不寫默認get
    callback=self.處理函數,
    priority=0 # 通常可不寫
)
```

### 參數說明

- `url`: 請求地址
- `method`: 請求方法(get/post) [post方法使用]()
- `callback`: 回調函數,指定處理響應的函數
- `priority`: 優先級,數字越小越優先,不設置則默認使用Stack順序

## URL 拼接

使用 `resp.urljoin(href)` 自動將相對地址拼接爲絕對地址

## 多次請求

```python
def parse(self, resp):
    for href in hrefs:
        ...
        yield scrapy.Request(
            url=href,
            callback=self.review
        )
def review(self, resp):
    print(resp.url)
```

如果在一個 `for` 循環中有多次 `yield`  
在 `callback` 當中執行時, 會按照順序從後往前執行  
如圖所示
```
Stack(堆疊)
|  E  |  ← 最後放進去的
|  D  |
|  C  |
|  B  |
|  A  |  ← 最先放進去的
└─────┘
取出時從上往下取出
```
### 使用enumerate 與priority

可以通過 `enumerate` 添加索引:

```python
for i, href in enumerate(hrefs):
    ...
```

`enumerate` 爲循環中的每個元素添加索引編號:
- `a.jpg, b.jpg, c.jpg` → `0 a.jpg, 1 b.jpg, 2 c.jpg`

### 配合priority 實現先入先出

使用 `priority=-i` (負索引號):

```python
for i, href in enumerate(hrefs):
    yield scrapy.Request(
        url=resp.urljoin(href),
        callback=self.parse,
        priority=-i
    )
```

效果: 先放入的請求優先取出, 後放入的請求後取出. 即 **先入先出 後入後出**

## 多頁請求
尋找下一頁按鈕的href 地址  
在回調函數中再次 `yield scrapy.Request()` 即可實現翻頁

## 數據流向

```
parse (發送請求) → yield Request → callback (處理響應)
```

## 重點說明

- 通過 `yield scrapy.Request()` 發送新請求
- `callback` 指定處理響應的函數, 默認爲 `parse`
- 使用 `resp.urljoin()` 拼接相對路徑
- 可在回調函數中繼續yield Request 實現翻頁
