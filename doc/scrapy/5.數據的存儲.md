# 數據的存儲

## 爲何使用open_spider 和close_spider

在 `process_item` 中使用 `with open()` 會導致每次處理數據時都反復開啓和關閉文件, 影響性能

## open_spider 方法

```python
def open_spider(self, spider):
    # 在爬蟲開始時自動執行
    self.f = open("文件名", mode="a", encoding="utf-8")
```

- 在爬蟲啓動時自動調用
- 只用於初始化資源（如打開文件）
- 只執行一次

## close_spider 方法

```python
def close_spider(self, spider):
    # 在爬蟲結束時自動執行
    if self.f:
        self.f.close()
```

- 在爬蟲結束時自動調用
- 只用於清理資源（如關閉文件）
- 只執行一次

## process_item 中的使用

```python
def process_item(self, item, spider):
    # 直接使用 self.f 寫入, 無需再次打開
    self.f.write(數據)
    return item
```

## 數據流向

```
open_spider (初始化文件) → process_item (循環寫入) → close_spider (關閉文件)
```

## 重點說明

- `open_spider`: 爬蟲開始時執行一次, 初始化文件對象
- `close_spider`: 爬蟲結束時執行一次, 關閉文件對象
- 文件對象保存爲 `self.變量名`, 供整個pipeline使用
- 避免在 `process_item` 中反復開啓和關閉文件
