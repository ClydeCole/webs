# 刪除數據

## delete 語句

刪除數據使用 SQL 的 `delete from` 語句

需要指定表名和刪除條件

```sql
delete from stu where sno = 11
```

## where 條件的必要性

delete 和 update 一樣, 不加 where 條件會清空整張表

這條規則在任何數據庫中都適用

在生產環境中刪除數據時, 通常會先查詢確認, 或者開啓事務後先執行再決定是提交還是回滾

## 軟刪除的觀念

在實際項目中, 很少真正物理刪除數據

更常見的做法是軟刪除, 也就是給表加一個狀態字段, 比如 `is_deleted`

刪除時只是把這個字段從 0 改成 1

查詢時默認過濾掉 `is_deleted = 1` 的數據

這樣做的好處是數據不會真正丟失, 隨時可以恢復

軟刪除的表結構通常是這樣的

```sql
alter table stu add is_deleted tinyint default 0
```

查詢時加上 `where is_deleted = 0` 即可

## 獲取影響行數

delete 的 `execute` 返回值同樣表示受影響的行數

可以用它來判斷是否成功刪除了數據

```python
count = cursor.execute(sql)
print(count)  # 刪除了幾條數據
```

## 事務要求

delete 也是修改操作, 需要提交事務

```python
cursor.execute(sql)
conn.commit()
```

在刪除的場景下, 事務尤其有用

可以先執行 delete, 確認刪除的數據正確後再提交

如果刪錯了, 直接回滾即可恢復
