# 查詢

## select 語句

查詢使用 SQL 的 `select` 語句

與增刪改不同, 查詢**不需要**提交事務, 因爲查詢不修改數據

```sql
select * from stu
```

## execute 的返回值

執行查詢時, `execute` 返回的是這個查詢匹配到的數據行數

而不是數據本身

實際的查詢結果需要通過遊標提供的方法來提取

```python
count = cursor.execute(sql)
print(count)  # 這個數字表示有多少條數據符合條件
```

## fetchone

`fetchone` 從遊標中提取一條數據

每調用一次, 遊標的指針就往後移動一位

多次調用 fetchone 會依次返回後續的每一條數據, 直到沒有數據時返回 None

適用於隻需要一條結果的場景, 比如根據主鍵查詢

```python
one = cursor.fetchone()
```

## fetchall

`fetchall` 一次性提取遊標中的全部數據

返回的是一個列表, 列表中的每一項是一條數據

當數據量很大時, fetchall 可能會佔用較多內存

適用於數據量不大且需要全部處理的場景

```python
all_data = cursor.fetchall()
for row in all_data:
    print(row)
```

## fetchmany

`fetchmany` 一次提取指定數量的數據

需要在參數中傳入要提取的條數

適用於分頁或分批處理的場景, 可以控制每次加載到內存的數據量

```python
some_data = cursor.fetchmany(3)
```

## 三種方法的關係

三種提取方法是共用同一個遊標位置的

也就是說, 如果先 fetchone 拿了一條, 接着 fetchmany 會從第二條開始拿

再 fetchall 會拿剩下的全部

因此在使用時要注意提取的順序, 避免取到意料之外的數據

## 查詢的完整流程

查詢不需要事務, 所以流程比增刪改更簡單

創建連接 → 創建遊標 → 執行 SQL → 提取數據 → 關閉遊標 → 關閉連接

```python
cursor.execute(sql)
result = cursor.fetchall()
cursor.close()
conn.close()
```
