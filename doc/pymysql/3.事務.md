# 事務

## 什麼是事務

事務是一組 SQL 操作的集合, 要麼全部執行成功, 要麼全部不執行

pymysql 默認開啓事務, 執行修改類SQL 後不會立即生效, 需要手動確認

## 爲什麼需要事務

數據庫操作可能在中途發生意外

例如批量插入數據時, 插入到一半程序崩潰了  
如果沒有事務機制, 已經插入的數據會殘留在數據庫中, 造成數據不完整

有了事務, 就可以在出錯時撤銷所有操作, 讓數據庫恢復到執行前的狀態

## commit

`commit` 用於確認事務, 將這段時間內的所有修改永久寫入數據庫

一旦提交, 就無法再回滾  
提交時機一般是在所有SQL 都執行成功之後

```python
conn.commit()
```

## rollback

`rollback` 用於撤銷事務, 將數據庫恢復到本次事務開始前的狀態  
通常放在異常處理中, 當某條 SQL 執行失敗時調用

```python
conn.rollback()
```

## try-finally 的使用

標準的事務處理模式是配合 `try-finally` 使用

推薦在 `try` 中執行SQL, 成功後提交  
出錯時在`except` 中回滾  
最終在`finally` 中關閉資源  

這樣無論程序是否出錯, 都能保證資源被正確釋放

```python
try:
    cursor.execute(sql)
    conn.commit()
except:
    conn.rollback()
finally:
    if cursor:
        cursor.close()
    if conn
        conn.close()
```

## 事務的邊界

- 事務從 `execute` 開始
- 遇到 `commit` 或 `rollback` 結束
- 結束後執行 `finally` 關閉遊標和連接

查詢操作不需要事務, 因爲查詢不修改數據

只有 insert、update、delete 這類修改操作才需要考慮事務
