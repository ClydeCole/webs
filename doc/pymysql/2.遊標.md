# 遊標

## 遊標的作用
遊標是數據庫操作中的核心概念  
連接對象本身不能直接執行SQL 語句, 需要通過 **遊標** 來完成  
遊標負責將SQL 語句發送給數據庫, 並接收數據庫返回的結果

## 默認遊標

不帶任何參數創建的遊標, 返回的數據格式是**元組**

```python
cursor = conn.cursor()
```

執行查詢後, 取出來的一條數據是一個元組, 多條數據是元組組成的列表  
元組的缺點是隻能通過下標訪問字段, 無法直接通過字段名獲取, 代碼可讀性較差

## DictCursor

爲了解決元組可讀性的問題, 可以使用 `DictCursor`

```python
from pymysql.cursors import DictCursor
cursor = conn.cursor(DictCursor)
```

DictCursor 返回的數據是**字典**格式

一條數據是一個字典, 鍵是字段名, 值是對應的字段值  
多條數據就是字典組成的列表

這樣就可以直接通過字段名來獲取數據, 代碼更清晰, 也更不容易出錯
例如 `row["name"]` 比 `row[0]` 的含義要明確得多

## 遊標的迭代特性

遊標存儲的數據具有類似迭代器的特性, 不是一次性全部加載到內存, 而是一條一條地取出  
每次調用提取方法, 都會從當前位置往後移動  
已經取過的數據不會再次出現  
這意味着如果需要重新遍歷, 必須重新執行一次查詢

## 遊標的生命週期

遊標在使用完畢後建議關閉
開多個遊標而不關閉, 可能會消耗數據庫資源

```python
if cursor:
    cursor.close()
```
