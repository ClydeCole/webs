# pymysql 數據庫的基本用法

## 連接

### 創建一個連接

創建連接只需要導入pymysql 使用:

```python
import pymysql
conn = pymysql.connect(
    host="localhost",
    ...
)
```

即可創建連接

### 連接參數

要連接MySQL 數據庫, 需要提供一些信息

- **host**: 數據庫所在服務器的地址, 本地通常爲 `localhost`
- **user**: 數據庫用戶名, 預設爲 `root`
- **password**: 對應用戶的密碼
- **database**: 要操作的數據庫名稱
- **port**: MySQL 默認端口是 `3306`

連接成功後, pymysql 會返回一個連接對象, 我們賦值給了 `conn`  
所有後續的數據庫操作都需要依靠這個對象來完成

## [遊標](2.遊標.md)

遊標是數據庫操作中的核心概念  
連接對象本身不能直接執行SQL 語句, 需要通過 **遊標** 來完成  

### 遊標的創建

創建遊標只需要在連接對象 `conn` 上調用 `cursor` 方法即可

```python
cursor = conn.cursor()
```

### DictCursor

不帶任何參數創建的遊標, 返回的數據格式是**元組**, 可讀性低  
爲了解決元組可讀性的問題, 可以使用 `DictCursor`

```python
from pymysql.cursors import DictCursor
cursor = conn.cursor(DictCursor)
```

之後遊標返回的數據將會使字典, 可讀性更高

## [事務](3.事務)

在pymysql 中執行修改類的SQL  
如 insert, update, delete 時
默認會開啓事務

### 提交數據

這意味着 SQL 雖然執行了, 但數據還沒有真正寫入數據庫  
必須手動提交事務, 數據纔會永久保存  

如果不想提交, 也可以選擇回滾, 撤銷本次操作

```python
conn.commit()    # 提交
conn.rollback()  # 回滾
```

### 關閉資源

數據庫操作完成後, **必須關閉連接!**
雖然 Python 的垃圾回收機制最終會處理  
但手動關閉是良好的編程習慣, 尤其是在長連接或高併發的場景下

```python
if conn
    conn.close()
```

### [try-finally](3.事務#try-finally 的使用)

推薦在 `try` 中執行SQL, 成功後提交  
出錯時在`except` 中回滾  
最終在`finally` 中關閉資源  

```python
try:
    cursor.execute(sql)
    conn.commit()
except:
    conn.rollback()
finally:
    if cursor:
        cursor.close()
    if conn
        conn.close()
```
